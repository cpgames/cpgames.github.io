ChillPill Coding Philosophy

  • Composition over inheritance.
  • Program to interface, not implementation.
  • Comment API, not implementation. Keep comments in implementation to the minimum, if implementation needs comments, write cleaner code.
  • Explicit variable names, no abbreviations. If variable/function/class name is too long, consider shorter alternatives.
  • Make API as generic and abstract as possible.
  • API should strive for thin implementation layer.
